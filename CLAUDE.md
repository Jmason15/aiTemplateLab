# aiTemplateLab вЂ” Claude Context

## What This App Does

A single-page browser app for creating, managing, and exporting structured AI prompt templates. Templates follow a 7-section schema (Objective, Actor, Context, Inputs, Constraints, Outputs, Success Criteria) and are persisted to `localStorage`. The build step compiles everything into one self-contained HTML file with no external dependencies.

The app always opens on the home screen with no template selected. The sidebar shows all template groups as collapsible sections with a search bar at the top. A Windows-style menu bar (File | Groups | Templates | Help) replaces the old dropdown/kebab pattern.

---

## Build System

```bash
node build.js          # produces dist/aiTemplateLab.html
npm run build          # same, via package.json script
```

`build.js` performs four steps:
1. **Generate prompt data** вЂ” reads `main/config/workspaces.json`, loads each referenced `main/Prompts/*.json` file, and builds the `preloadedPrompts` / `preloadedWorkspaces` / `preloadedConfig` JS string in memory.
2. **Resolve HTML partials** вЂ” replaces `<!-- #include "path/to/partial.html" -->` directives with file contents.
3. **Inline CSS** вЂ” replaces all `<link rel="stylesheet">` tags with a single `<style>` block containing all CSS files concatenated in order.
4. **Inline JS** вЂ” replaces each `<script src="js/...">` tag with an inline `<script>` block. `preloadedPrompts.js` is special: it is generated in memory (step 1) and never exists on disk.

**The dist file is the only deployable artifact.** After any source change, rebuild and commit both the source and `dist/aiTemplateLab.html` together.

## Adding a Prompt Template

1. Create a JSON file in `main/Prompts/` following the data model below. The file can be a plain object `{}` or an array `[{}, {}]`.
2. Add the filename to `main/config/workspaces.json` under the appropriate workspace.
3. Run `node build.js`.

**`workspaces.json` shape:**
```json
{
    "workspaces": {
        "Template Lab": ["templateBuilder.json", "groupGenerator.json", "workflowGenerator.json"],
        "Example Templates": ["jiraStory.json", "codeCleaner.json"]
    }
}
```

No `defaultWorkspace` or `defaultTemplate` вЂ” the app always opens on the home screen. You do not need to touch any JS files to add a new prompt.

## Data Model

Every template has this shape:

```json
{
  "id": "readable-slug",
  "name": "Template name",
  "description": "Short description shown in the sidebar info bar",
  "objective": "What this prompt accomplishes",
  "actor": "AI persona/role",
  "context": "Background context for the AI",
  "example": "Plain-English walkthrough shown in the 'See an Example' section (optional)",
  "inputs": [
    { "name": "fieldName", "description": "what to provide", "placeholder": "example value shown in the textarea" }
  ],
  "constraints": ["rule 1", "rule 2"],
  "outputs": [
    { "name": "outputName", "type": "string", "description": "what comes back" }
  ],
  "success": ["criterion 1", "criterion 2"]
}
```

**localStorage keys:**
- `templateGroups` вЂ” `{ [groupName]: prompt[] }`
- `templateGroupHistory` вЂ” `{ [groupName]: inputHistory[] }`
- `currentTemplateGroup` вЂ” active group name string

## State Management

State is global вЂ” no framework. The authoritative variables are declared as `let` in `state.js` and are accessible to all later-loaded scripts.

| Variable | Declared in | Purpose |
|---|---|---|
| `prompts` | state.js | Flat working array of templates in the active group |
| `currentPromptId` | state.js | ID of the currently selected template |
| `currentTemplateGroup` | state.js | Active workspace/group name |
| `environment` | state.js | `{ templateGroups: {}, history: {} }` вЂ” full authoritative store |

**Dual-write pattern:** `prompts[]` (flat working array) and `environment.templateGroups[currentTemplateGroup][]` (authoritative source) must always be kept in sync. Any mutation should update both and call `savePromptsToLocalStorage()`.

**`window.*` sync:** `editPrompt.js` reads state via `window.*` (e.g. `window.currentPromptId`) because it was written independently of `state.js`. `syncWindowState()` in `state.js` keeps `window.*` mirrors in sync. Call it after any state mutation.

## Key Patterns

**`normalizePrompt()`** вЂ” guards against legacy data where `constraints` or `success` entries were stored as objects instead of strings. Always call this when loading prompts from storage or files: `prompts.map(normalizePrompt)`.

**Debounced auto-save** вЂ” in `editPrompt.js`, every field input event fires `debouncedSaveCurrentPrompt` (200ms delay). `saveCurrentPrompt()` uses counter-based iteration (`for i = 1..inputCounter`) rather than `querySelectorAll` so it correctly skips elements that were deleted and removed from the DOM.

**`window.regenerateOutput`** вЂ” defined in `promptCrud.js`. Called by `editPrompt.js` on every field change. Guards against running while the edit screen is not active.

**`resetToPreloaded()`** вЂ” iterates `window.preloadedWorkspaces` dynamically (not hardcoded names), so adding new workspaces to `workspaces.json` requires no code change.

**`window.showHelpModal(key)`** вЂ” exposed by `workspaceManager.js`. Accepts a key matching the `helpContent` map (e.g. `'help-what-is-template'`). Used by home screen chips and the Help menu to open the help modal without duplicating content.

**Sidebar search** вЂ” `renderPromptsList()` reads `#sidebar-search` value on every call. When a query is present it filters all groups and shows only matching tiles with all sections expanded. When empty it renders the normal grouped view.

## UI Structure

**App bar** вЂ” fixed top bar with the app name (home button) and a Windows-style menu bar: File | Groups | Templates | Help. Each top-level button toggles its dropdown; hovering switches between open dropdowns.

**Sidebar** вЂ” fixed left panel containing: search bar, collapsible group sections (all groups shown, active group open by default), and two action buttons at the bottom (green **Import Template From AI**, grey **New Blank Template**).

**Main content** вЂ” info bar (name, description, See an Example, Show Details) + tab bar (Use Template | Edit Template | Template History | Output) + tab content panels.

**Home screen** вЂ” always shown on startup and when clicking the app name. Contains a hero section, 3-step how-to, and four collapsible accordions with help chip links.

## Adding a New Template Section

To add a new section to the prompt schema (e.g., "Examples"):

1. Add HTML for the section in `main/index.html` (edit form area) and any relevant partial.
2. Add an `addExample()` function in `editPrompt.js` following the `addTextareaField()` pattern.
3. Update `saveCurrentPrompt()` in `editPrompt.js` to read the new fields.
4. Update `viewPrompt()` in `promptCrud.js` to display the new fields.
5. Update `generateViewPrompt()` in `promptOutput.js` to include them in JSON output.
6. Rebuild: `node build.js`.

## CSS File Order

CSS files must be added to `CSS_FILES_ORDERED` in `build.js` **and** to `<link>` tags in `main/index.html` in the same order. `variables.css` must always be first (defines all custom properties used by other files).

## Gotchas

- `preloadedPrompts.js` is generated in memory by `build.js` вЂ” never create or commit this file manually.
- There is no dev server and no test suite. Open `main/index.html` directly in a browser for local dev; rebuild to produce the dist file.
- `localStorage` is per-origin. Testing `main/index.html` via `file://` and `dist/aiTemplateLab.html` share the same origin, so they share localStorage state.
- The app has no undo. Deleting a template is permanent.
- No linter or formatter is configured. Be consistent with the surrounding code style.
- `base.css` applies `details { background: #f0f0f0; padding: 8px 12px }` globally. Override per-component (e.g. `.group-section` in sidebar.css uses `background: transparent; padding: 0`).
- The sidebar's generic `#sidebar button` rule overrides btn classes вЂ” use ID-specific rules with `!important` when a sidebar button needs a distinct colour (e.g. `#import-json-btn` for the green import button).
