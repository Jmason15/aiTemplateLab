# aiTemplateLab

A browser-based tool for building, managing, and reusing structured AI prompt templates. Fill in your inputs, click **Create Prompt**, and paste the result into your favorite AI tool — every time.

---

## What It Does

aiTemplateLab helps you stop rewriting the same prompts from scratch. Build a template once with a consistent structure, fill in your details each time you need it, and get a polished, ready-to-use prompt instantly.

Each template is built around a 7-section schema:

- **Objective** — what the prompt should accomplish
- **Actor** — the AI persona or role
- **Context** — background the AI needs
- **Inputs** — fields you fill in each time you use the template
- **Constraints** — rules the AI must follow
- **Outputs** — what the AI should return and in what format
- **Success Criteria** — how to evaluate a good response

---

## Getting Started

**No install required.** Download `dist/aiTemplateLab.html` and open it in any modern browser. Everything runs locally — no server, no account, no data sent anywhere.

---

## Using a Template

1. Pick a template from the sidebar
2. Fill in the input fields on the **Use Template** tab
3. Click **Create Prompt** — the prompt is automatically copied to your clipboard
4. Paste it into your favorite AI tool
5. Copy the AI's response and use it — or import it back (see Template Lab below)

Your input values are saved to history automatically. Access previous runs from the **Template History** tab.

---

## Building Your Template Library

The **Template Lab** group comes with three tools for generating new templates using your AI tool:

### Template Builder
Describe what you want a prompt to do. Your AI returns a complete, ready-to-import template.

### Template Group Generator
Enter your job title or area of work. Your AI generates a full set of templates covering every task you'd regularly need AI help with.

### Workflow Generator
Describe a goal and your AI breaks it into a step-by-step sequence of templates — one per stage, where each step feeds into the next.

**To import templates generated by any of these tools:**
1. Run the template in your AI tool and copy the response
2. Click the green **Import Template From AI** button in the sidebar
3. Paste the response and click Import — your new templates appear instantly

## Managing Templates

### Edit a Template
Click the **Edit Template** tab while a template is selected. Changes auto-save as you type.

### Create a Template from Scratch
Click **New Blank Template** in the sidebar or go to **Templates → New** in the menu bar.

### Organize with Template Groups
Templates are organized into groups. Groups are collapsible in the sidebar — click a group name to expand it. Use **Groups** in the menu bar to create, save, load, or delete groups.

### Search
Use the search bar at the top of the sidebar to filter templates by name or description across all groups.

### Import / Export
- **Import Template From AI** (green sidebar button) — paste in a template generated by your AI tool
- **Templates → Export** — download selected templates as a file to share or back up
- **File → Save** — save your entire workspace (all groups and history) as a single file
- **File → Load** — restore a previously saved workspace

---

## Data Storage

Templates and history are saved automatically to your browser's local storage. They persist across page refreshes and browser restarts on the same device.

Local storage is tied to the specific browser and computer you are using. Use **File → Save** regularly to back up your work — if you clear your browser data or switch devices, unsaved templates cannot be recovered.

## Building from Source

```bash
npm run build
# produces dist/aiTemplateLab.html
```

Source files live in `main/`. The build step inlines all CSS and JS into a single self-contained HTML file. See [CLAUDE.md](CLAUDE.md) for full developer documentation.

---

## Adding Prompt Templates

1. Create a JSON file in `main/Prompts/` following the template schema
2. Reference it in `main/config/workspaces.json` under the appropriate workspace
3. Run `npm run build`

No JavaScript changes needed. See [CLAUDE.md](CLAUDE.md) for the full data model.
