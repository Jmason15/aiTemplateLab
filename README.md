# aiTemplateLab

A browser-based tool for building, managing, and reusing structured AI prompt templates. Fill in your inputs, click **Copy Prompt**, and paste the result into your favorite AI tool — every time.

---

## What It Does

aiTemplateLab helps you stop rewriting the same prompts from scratch. Build a template once with a consistent structure, fill in your details each time you need it, and get a polished, ready-to-use prompt instantly.

Each template follows a 7-section schema:

- **Objective** — what the prompt should accomplish
- **Actor** — the AI persona or role
- **Context** — background the AI needs
- **Inputs** — fields you fill in each time you use the template
- **Constraints** — rules the AI must follow
- **Outputs** — what the AI should return and in what format
- **Success Criteria** — how to evaluate a good response

## Getting Started

**No install required.** Download `dist/aiTemplateLab.html` and open it in any modern browser. Everything runs locally — no server, no account, no data sent anywhere.

---

## Using a Template

1. Pick a template from the sidebar
2. Fill in the input fields
3. Click **Copy Prompt** — the prompt is copied to your clipboard
4. Paste it into your favorite AI tool

Your input values are saved to history automatically. Access previous runs from the history dropdown on the template view.

## Building Your Template Library

The **Template Lab** group includes three tools for generating new templates with your AI:

### Template Builder
Describe what you want a prompt to do. Your AI returns a complete, ready-to-import template.

### Template Group Generator
Enter your job title or area of work. Your AI generates a full set of templates covering every task you'd regularly need AI help with.

### Workflow Generator
Describe a goal and your AI breaks it into a step-by-step sequence of templates — one per stage, where each step feeds into the next.

**To import templates generated by any of these tools:**
1. Run the template in your AI and copy the response
2. Click **Import** at the bottom of the sidebar
3. Drop or paste the file — templates appear instantly

## Managing Templates

### Using the Menu Bar
A Windows-style menu bar (File | Groups | Templates | Help) sits at the top of the app:

- **File** — Save Workspace, Load Workspace, Reset, Download Latest
- **Groups** — New, Export, Delete
- **Templates** — New Blank, Export
- **Help** — help topics and guidance

### Edit a Template
Select a template, then click **Edit** (pencil icon). Changes auto-save as you type — there is no Save button.

### Build a Template from Scratch
Click the green **Build a New Template** tile at the top of the sidebar to use the guided builder screen.

### Organize with Groups
Templates are organized into collapsible groups in the sidebar. Use **Groups** in the menu bar to create, export, or delete groups.

### Search
Use the search bar at the top of the sidebar to filter templates by name or description across all groups.

### Import / Export
- **Import** (sidebar footer button) — drop in a template or group JSON file
- **Export / Share** (sidebar footer button) — download or share this template, this group, or everything
- **File → Save Workspace** — back up your entire workspace (all groups and history)
- **File → Load Workspace** — restore a previously saved workspace

---

## Data Storage

Templates and history are saved automatically to your browser's local storage. They persist across page refreshes and browser restarts on the same device.

Local storage is tied to the specific browser and computer you are using. Use **File → Save Workspace** regularly to back up your work — if you clear your browser data or switch devices, unsaved templates cannot be recovered.

## Building from Source

```bash
npm run build
# produces dist/aiTemplateLab.html and docs/index.html
```

Source files live in `main/`. The build step inlines all CSS and JS into a single self-contained HTML file with no external dependencies.

### Adding a Prompt Template

1. Create a JSON file in `main/Prompts/`
2. Reference it in `main/config/workspaces.json` under the appropriate workspace
3. Run `npm run build`

No JavaScript changes needed. See [CLAUDE.md](CLAUDE.md) for the full data model and developer documentation.

### Releasing

```bash
git tag release_YYYY_MM_DD
git push origin release_YYYY_MM_DD
```

GitHub Actions will build, deploy to GitHub Pages, and attach `dist/aiTemplateLab.html` to a GitHub Release automatically.
