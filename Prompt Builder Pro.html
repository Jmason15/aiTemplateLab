      Prompt Info
      
                        
                        
                        
                        
                        
                    
      

                    
      
                        
      1. Objective (Task)
      
                        
      One short sentence that states the main job you want the model to do.
      
                        
                    
      

                    
      
                        
      2. Actor (Role / Persona)
      
                        
      The hat the model wears: background, expertise, and point of view.
      
                        
                    
      

                    
      
                        
      3. Context
      
                        
      Brief background the model needs to behave correctly in your domain.
      
                        
                    
      

                    
      
                        
      4. Input (Schema / Fields)
      
                        
      The structured data the model will receive.
      
                        
      
                        
                    
      

                    
      
                        
      5. Constraints (Rules)
      
                        
      The do/don't list that shapes style, scope, and behavior.
      
                        
      
                        
                    
      

                    
      
                        
      6. Output (Format / Schema)
      
                        
      The exact shape of the answer the model should return. Define multiple output properties.
      
                        
      
                        
                    
      

                    
      
                        
      7. Success (Evaluation Criteria)
      
                        
      How you'll judge whether a response is "good enough."
