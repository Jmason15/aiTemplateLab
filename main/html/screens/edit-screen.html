      Prompt Info
      What is this? The name and description help you and others quickly identify what this prompt is for.
      Why is it important? Clear names and descriptions make it easy to find and reuse prompts for different business or technical needs.
      Business Example: "Quarterly Sales Report" or "Summarize meeting notes for executives."
      Technical Example: "Generate a REST API endpoint for user authentication." or "Create a SQL query to aggregate sales data."
      1. Objective
      What is this? The objective states the main goal of the prompt.
      Why is it important? A clear objective ensures the output matches your expectations and business goals.
      Business Example: "Summarize the main points from a business meeting in plain language."
      Technical Example: "Document the steps to deploy a Node.js application to AWS."
      2. Actor
      What is this? The actor is the person or role performing the task described by the prompt.
      Why is it important? Defining the actor helps tailor the output to the right audience or expertise level.
      Business Example: "You are a business analyst preparing a summary for the CEO."
      Technical Example: "You are a DevOps engineer automating deployment scripts."
      3. Context
      What is this? The context provides background information or the situation for the prompt.
      Why is it important? Good context ensures the output is relevant and fits the intended use case.
      Business Example: "This summary will be shared with the executive team after each meeting."
      Technical Example: "This script will be used by developers to automate database backups."
      4. Input
      What is this? Inputs are the information the user must provide for the prompt to work.
      Why is it important? Listing inputs ensures nothing is missed and the output is accurate.
      Business Example: "Meeting notes, list of attendees, action items."
      Technical Example: "API endpoint URL, request parameters, authentication token."
      5. Constraints
      What is this? Constraints are rules or requirements the output must follow.
      Why is it important? Constraints help ensure the results meet business standards or technical requirements.
      Business Example: "Keep the summary under 200 words. Avoid technical jargon."
      Technical Example: "Response time must be under 2 seconds. Use only GET and POST methods."
      6. Output
      What is this? The output describes what the result should look like.
      Why is it important? Defining the output helps you get exactly what you need, in the right format.
      Business Example: "A short summary with key decisions and next steps."
      Technical Example: "A JSON object with user data and status code."
      7. Success
      What is this? Success criteria define how you know the output is good.
      Why is it important? Clear success criteria make it easy to check if the results meet your needs.
      Business Example: "The summary is clear, easy to read, and covers all important points."
      Technical Example: "All unit tests pass and the API returns the expected response."
